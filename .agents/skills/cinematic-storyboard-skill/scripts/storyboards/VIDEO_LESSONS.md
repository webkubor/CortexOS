# 沸腾之雪 · 视频脚本生成全规则

> **本文件是项目核心价值产出。** 任何人/AI写视频脚本前必读，违反=废片。
> 不再分散到其他文件，所有视频生成的规则、美学、教训统一收在此处。

---

## 一、时间分配（15秒）

| 时段 | 秒数 | 做什么 | 禁止 |
|------|------|--------|------|
| 起势 | 0-2s | 首次接触，交代环境 | 空拍摆pose，超过2s |
| 激战 | 2-13s | 全部精华 | 超过1s无攻防交换 |
| 收尾 | 13-15s | 定格/对视，1秒内结束 | 文艺收尾、缓慢走远 |

## 二、镜头美学

### 2.1 运镜选择（每段只选一种，写在动作之前）

| 场景 | 运镜 | 原因 |
|------|------|------|
| 对穿抢位 | 中景侧跟 | 看双方位移和距离 |
| 轻功腾空 | 低机位仰拍 | 突出高度和速度 |
| 近身破招 | 贴身短焦 | 武器距离内打击细节 |
| 群战调度 | 高机位俯拍 | 看清多人站位和阵型 |
| 史诗高潮 | 大远景俯拍 | 宏大尺度，7-10s区间用一次 |
| 收尾定格 | 缓拉中景 | 自然收束 |

### 2.2 切镜频率
- 15秒片 3-5次切镜，不超过7次
- 每次切镜必须有信息增量
- 同一段内（2-3秒）保持同一运镜

### 2.3 史诗镜头
- 全片最多1次，1-2秒，必须在7-10s区间
- 必须包含人物+战斗痕迹，禁止纯环境空镜
- 禁止用于起势和收尾

### 2.4 特写配额

| 类型 | 配额 | 位置 |
|------|------|------|
| 战斗特写 | 占比≥60% | 打斗中间（武器接触瞬间） |
| 环境特写 | 由动作触发 | 必须写触发原因 |
| 表情特写 | ≤1次，≤0.3s | 打斗中间 |
| 慢镜头 | ≤1次，0.2-0.35s | 打斗中间（关键打击瞬间） |

**慢镜头只用于**：武器接触皮肤、刀锋擦过血线、剑尖点破衣甲。极速回正。
**慢镜头禁止用于**：起势摆拍、收尾回眸、头发飘散。

### 2.5 反模式
- ❌ "snap zoom眼部特写瞳孔微缩" — 慢镜头摆拍
- ❌ "缓缓抬眼/剑归鞘/风吹衣袂" — 文艺收尾
- ❌ "环绕360°展示身姿" — 摆pose
- ❌ "镜头环拍/急甩/跳切堆砌" — 运镜堆砌
- ❌ "火焰分裂高光帧" — 纯物品特写没人

## 三、轻功描写规范

### 3.1 核心原则：必须写"离地"

❌ 错误写法 → AI会怎样理解：
- "贴地滑行" → 蛤蟆趴在地上往前爬
- "蹬墙奔跑" → 普通跑墙，没有腾空
- "滑步/垫步/切步" → 普通走路变体

✅ 正确写法：
- 身体凌空离地30cm向前高速平移
- 双脚全程不触地面
- 身体水平悬停/足尖不触地
- 从起跳到落地每一帧脚不在地面上

### 3.2 动作链模板
```
脚点X物借力 → 身体离地Y厘米沿Z方向平移N米 → 落地时脚点Y物产生物理反馈
```

### 3.3 运镜配合
- 优先长镜头跟随 + 背景动态模糊
- 禁止复杂运镜（环绕360°/急甩）拍轻功，会暴露肢体瑕疵

## 四、动作尺度与力量感

### 4.1 天榜级高手对打的动作尺度
两个天榜高手交锋不是在角落里格挡挠痒，必须体现**空间感和力量感**：

- **位移尺度**：每次交锋的位移至少5-10米，不是1-2米的小碎步
- **环境破坏**：不是踩裂一块地，是一整片区域被气浪夷平（树木连片折断、地面犁出十几米沟痕、积雪被推飞形成真空带）
- **交锋距离**：两人交锋时应该有明显的冲撞和分离，不是贴在一起缠斗
- **力量外溢**：每一招的余波必须影响周围环境（风压刮倒远处的人、气浪震碎远处的东西、地面持续龟裂扩散）

### 4.2 对比模板

❌ 天榜高手打起来像这样：
"两人后退两步""掌心碰撞爆开气雾""滑退四米"

✅ 天榜高手打起来应该像这样：
"掌心碰撞瞬间气浪呈环形向外扩散二十米，沿途枯树连片从根部折断树冠倾泻积雪，地面从碰撞点向外犁出扇形龟裂带，雪花被震成粉末悬浮在十米高空，远处倒地的北荒兵被余波推出数米"

### 4.3 密度控制
- **每段（2-3秒）只写1-2个核心动作**
- 多了AI平均分配注意力，全糊
- 宁可少而精，不要多而烂

### 4.4 位移必须写
- 每个动作附带：从哪到哪、横移几米、向前推进几米
- 起点和终点都要有
- 移动方式具体：凌空平移/蹬墙弹射/借力变向
- ❌ "她向前冲" → ✅ "她从巷口凌空平移到巷尾7米处"

### 4.5 禁止文学修辞
- 全部用物理量：身体部位+方向+力度+接触点+位移
- ❌ "如白鹤掠水/干净到像写字/猫式落地"
- ✅ "右脚蹬竹竿向左45°弹起3米"

## 五、武器物理

### 5.1 武器锁死
- 全程不变：长度/材质/不断裂/不变形/不脱手/不互换
- 主从关系锁定（左手什么右手什么）

### 5.2 武器脱手规则
- 轨迹必须符合物理（重力+抛物线+旋转衰减）
- ❌ 武器飞出后精准命中目标（弓箭除外）
- ❌ 武器脱手做花哨的事（劈断灯笼、削断竹竿）
- ✅ 刀脱手落地/刀飞出数米钉入地面

### 5.3 弓手/弩手
- 必须写弹道：从哪射出→经过什么→钉在哪
- 近距离（<5米）禁止"完整拉弓实射"
- 闪避箭矢必须交代：箭从哪来→怎么躲→钉在哪→钉点特写

---

## 六、敌人调度

### 6.1 站位必须写
- 写了多少敌人就必须写多少站位和动作
- ❌ "黑影从深处浮现/刀手压上"
- ✅ "3名刀手从左前方7米处呈扇形逼来，2名弩手从右侧高处射箭"

### 6.2 禁止回合制
- 高手打菜鸟是"一招带一片"，不是一个一个杀
- 多人同时压迫→主角破阵→敌人互相挡路/阵型崩溃
- ❌ 主角打一个，其余人站着等

---

## 七、收尾规则

- 15秒片收尾不超过1秒
- ❌ "缓缓走入枯林深处""没入风雪""雨声回归""雪风吹得猎猎作响"
- ✅ 对视/定格/一个动作结束
- 省下来的时间给打斗

---

## 八、人物与场景写法

### 8.1 场景必须写尺度
- ❌ "枯林雪地" → AI生成狭小空间
- ✅ "北荒雪原枯林横向百米纵深五十米，枯树间距五到八米，天空低沉"
- 场景越大，AI给的动作空间越大，轻功和攻击力才体现得出来

### 8.2 人物不要分正反方
- ❌ "人物：XXX。敌方：YYY" → AI以为只有两个人
- ✅ "出场人物：慧明（参考图），萧烬弦（参考图），小兵一名，其余倒地的北荒兵作为背景"
- 直接列谁出场，不标正反派

### 8.3 交付格式

### 8.1 结构
```
场景：[地形+光照+天气+氛围]。人物：[参考图外貌着装]，武器+物理特性。敌方：[数量+武器+站位]。音效：[逗号分隔]。技术：[固定模板]。时间段1｜段名 [运镜]。[动作描述]。时间段2｜段名 [运镜]。[动作描述]。...最后一段｜段名 [运镜]。[动作描述，≤1秒]。
```

### 8.2 规则
- **纯文本，不分段不换行**
- **无卡点标注**
- 每段时间标题保留但紧跟运镜和动作
- 环境反馈融入动作描述，不单独写
- **不在视频脚本里写对白**（AI视频工具不支持，对白留给配音）

### 实录1：雨夜古巷（2026-03-22）
| 问题 | 原因 | 教训 |

## 十、武侠视觉物理标准（从已有成功模板对齐）

> 以下规律从慕北歌/苏梦城/欧阳狂徒/萧烬弦等已验证的分镜中提取。
> 每次写脚本必须对照，不符合物理逻辑 = AI 会生成乱七八糟的画面。

### 10.1 动作必须有物理来源（不能凭空发生）

| 模板示例 | 物理来源 |
|---------|---------|
| 萧烬弦从飞檐坠落→落地瞬步 | 重力势能→转化为水平动能 |
| 萧烬弦借肩膀后空翻 | 接触点反作用力 |
| 萧烬弦滞空满弓 | 顶部滞停瞬间（重力被弓弦张力抵消） |
| 欧阳狂徒跃起重砸地面 | 身体重力+腿部蹬力→刀刃动能 |
| 慕北歌站在弯曲冰竹上 | 冰竹弹性形变蓄力 |
| 慕北歌脚尖轻点雪面落地 | 用最小接触面积吸收冲击 |

**铁律**：任何人不能凭空移动/凭空飞到树上/凭空瞬移。每一个位移动作必须有明确的物理来源（重力、反弹、惯性、蹬力）。

### 10.2 杀人的方式是间接的（不是直接砍中）

| 模板示例 | 杀法 |
|---------|------|
| 慕北歌：横向一抹→三人咽喉红线并封冻 | 剑气余波冻住血管，不是刀口砍的 |
| 苏梦城：刺客直到此刻才缓缓倒地 | 枪劲渗透后延迟生效 |
| 欧阳狂徒：重甲碎裂+血液震成红雾 | 动能传递震碎护甲，不是刀刃切割 |
| 萧烬弦：寂灭指扣喉→迸发暗劲震退数米 | 指力穿透传递内伤 |

**铁律**：天榜高手杀人不是"砍一刀流一地血"。是通过内力/气劲/动能传递造成间接伤害，画面更干净更有逼格。

### 10.3 环境影响是余波不是原因（不能主动打环境）

| 模板示例 | 环境反应 |
|---------|---------|
| 慕北歌：脚尖轻点落地→十米内竹叶积雪同时崩碎 | 是剑气余波感应，不是他主动去打竹子 |
| 苏梦城：身后刺客才倒地+云海形成漩涡 | 气劲持续扩散，延迟生效 |
| 欧阳狂徒：刀势如旋风→瓦砾崩飞盔甲漫天 | 动能外溢带动周围，不是瞄准环境打 |
| 萧烬弦：箭贯穿回廊所有门窗+真空通道 | 箭矢动能穿过建筑物的连锁反应 |

**铁律**：高手打的是人，环境是余波。不能写"挥剑劈断竹竿""踢飞灯笼""削断树枝"这种主动攻击环境的动作，那不是高手，那是伐木工。

### 10.4 高手之间的交锋要有视觉层次

| 模板特点 | 说明 |
|---------|------|
| 慕北歌 | 极简，一抹就结束了，三人倒地。动作越少越显得强 |
| 苏梦城 | 一枪点眉心就结束战斗，但余波是刺客"才"倒地 |
| 欧阳狂徒 | 纯暴力美学，但每一刀都有物理重量感 |
| 萧烬弦 | 蓄力9秒（极静）→爆发3秒（极速），节奏反差 |

**铁律**：高手对打不是"你打一招我挡一招来回十几个回合"。是**短暂交锋→一招定胜负→余波扩散**。回合越少越显得强。

### 10.5 力度反馈必须有（不能轻飘飘举下手就倒了）

| ❌ 错误 | ✅ 正确 |
|---------|---------|
| "一掌拍开指法" | "掌心气浪与指力碰撞爆开白色气雾，两人同时向后滑退数米靴底在地面犁出深痕" |
| "玉笛刺中肩锁骨" | "玉笛刺中肩锁骨的瞬间箭矢钉入肩骨的冲击力将他向后推飞三米撞在石柱上石柱表面出现蛛网裂纹" |
| "一脚踢飞" | "右脚横扫正中手腕，踢击的冲击力将他整个人横向抛飞六米撞断两根枯树后摔在雪地" |
| "掌风推人滑退" | "掌风正面击中胸口的瞬间他身后空气被压缩形成冲击波，整个人倒飞出去将沿途三名士兵撞倒" |

**铁律**：每一次击中，必须写：
1. **碰撞瞬间的视觉效果**（气雾/火花/冲击波/裂纹）
2. **受击者的物理位移**（被推多远、撞到什么、撞了之后环境怎么坏）
3. **击中者与受击者的双方都有物理反应**（谁都没站稳、双方都滑退）

打中=力与力的碰撞，不是碰一下对方就倒。两个人都必须被力推开。

## 十一、Seedance 2.0 平台规则（必须遵守）

> 以下规则来自 Seedance 2.0 官方结构化提示词系统和社区实测经验。

### 11.1 Prompt 必须按 S-A-C-S 结构

Seedance 2.0 的标准 prompt 结构是固定的：

```
S（Subject 主体）：@图片1 角色参考，@图片2 场景参考
A（Action 动作）：每段一个核心动作，现在时态动词
C（Camera 镜头）：景别 + 运动 + 角度 + 镜头
S（Style 风格）：视觉锚点 + 光照 + 色调
约束：[负面约束词]
```

### 11.2 @图片引用系统

- 上传角色图 → `@图片1` 引用，不要写外貌描述
- 上传场景图 → `@图片2` 引用，不要写文字描述场景
- 上传武器图 → `@图片3` 引用，不要写武器外观
- **@引用 = AI严格跟随参考，文字描述 = AI自由发挥**

### 11.3 镜头用专业术语

wide shot (WS) / medium shot (MS) / close-up (CU) / dolly in-out / tracking shot / crane shot / low angle / high angle / handheld / gimbal / slow motion / snap zoom

### 11.4 负面约束（必须追加在prompt末尾）

```
约束：no extra characters, no crowd, no extra fingers, no extra limbs, no distortion, no deformed hands, no warped faces, no melting edges, no jump cuts, no lens flares, no floating UI, no text overlays, no watermarks, no blurry motion, no snap zooms, no whip pans, no oversaturated colors.
```

### 11.5 5秒分段迭代（复杂场景必用）

不要一次生成15秒复杂打斗。先5s × 3-5个变体 → 挑最好的 → 逐段延长。

### 11.6 Seedance 2.0 武侠打斗 Prompt 实例

```
@图片1 作为白衣侠客，@图片2 作为竹林雪夜场景。白衣侠客在竹林中高速穿梭闪避黑衣人刀锋，脚尖点竹竿凌空变向，玉笛横扫点倒三人，竹叶积雪在气浪中炸开如莲花绽放。crane shot 从高处俯拍竹林全景，侠客白衣在竹间穿梭如白色闪电，黑衣人成片倒伏。武侠动作片风格，35mm胶片颗粒，低调冷蓝色调，体积光穿透竹缝，动作流畅自然。约束：no extra characters, no crowd, no extra fingers, no distortion, no deformed hands, no jump cuts, no snap zooms, no blurry motion.
```
