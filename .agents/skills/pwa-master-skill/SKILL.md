# PWA 大师技能

此技能旨在将 Web 应用转变为稳健的 PWA，且不影响产品的稳定性。

当用户要求执行以下操作时使用：
- 为 Web 应用添加 PWA 支持
- 将现有应用迁移到 `vite-plugin-pwa` 或类似的设置
- 改进离线行为、安装提示或更新处理
- 添加或修复 `manifest.webmanifest`、图标或服务工作线程注册

请勿将此技能用于：
- 与 PWA 行为无关的通用前端重新设计
- 原生移动应用开发
- 广泛的性能调优（除非它直接影响离线或安装体验）

## 核心工作流

### 1. 首先检查应用
- 确定框架、构建工具、部署模型和当前的资源流水线
- 检查是否已存在清单、服务工作线程、图标集或安装提示逻辑
- 避免添加第二个竞争性的 PWA 技术栈

### 2. 定义正确的 PWA 行为
- 有意识地选择缓存行为，而不是缓存所有内容
- 将预缓存资源与运行时缓存分开
- 使更新行为明确，以免用户困在旧代码中
- 针对 iOS 和 Chromium 浏览器分别处理安装体验

### 3. 实现最小可靠界面
- 添加或修复 `manifest.webmanifest`
- 干净利落地注册服务工作线程
- 仅在应用确实需要时添加更新通知
- 使图标和主题元数据与应用保持一致

### 4. 验证面向用户的行为
- 确认离线回退或缓存行为按预期工作
- 确认更新流程不会导致重载循环或界面卡死
- 确认安装提示不会在不恰当的语境下显示
- 确认图标、主题颜色和独立运行行为连贯一致

## 加载规则

- 始终阅读 `references/pwa-expert.md`

## 约束

- 不要过度缓存动态或敏感数据
- 不要添加让人感觉像垃圾信息的安装提示
- 不要发布你未经过深思熟虑的服务工作线程
- 相比华而不实但脆弱的方案，更倾向于保守稳健的 PWA 方案
