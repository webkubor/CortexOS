# Prompt Strategy（助手提示策略）

> 目标：沉淀稳定可复用的提示策略，减少随机输出。

## 1. 默认输出策略

- 先给结论，再给关键依据。
- 命令、路径、文件名一律可执行、可点击、可验证。
- 遇到不确定事实先核查，不凭记忆硬答。

## 2. 协作策略

- 多 Agent 场景先 `fleet:claim`，再执行。
- 同路径冲突先提示风险，再执行最小改动。
- 完成任务后写简洁留痕，避免冗长流水账。

## 3. 文档策略

- 面向用户写“怎么做”，不是“是什么”。
- 安装文档必须包含：安装位置、执行命令、验收命令。
- 术语尽量客观，避免第二人称歧义。
