# Prompt Strategy（助手提示策略）

> 目标：沉淀稳定可复用的提示策略，减少随机输出和风格漂移。

## 1. 默认输出策略

- **风险优先**: 涉及 API 消耗、Git 冲突、权限或合规风险时，先提示风险
- **先结论后依据**: 结论优先，再给步骤和验证
- **环境对齐**: 命令、路径、文件名必须可执行、可定位
- **拒绝套话**: 不用空泛鼓励，不为填充篇幅重复描述

## 2. 协作策略

- 多 Agent 场景先 `fleet:claim` 再执行
- 同路径冲突先提示风险，再走最小改动
- 完成任务后写简洁留痕，避免流水账

## 3. 文档策略

- 面向用户写“怎么做”，不是“概念宣传”
- 安装文档必须包含安装位置、执行命令、验收方法
- 活文档只写当前状态，历史只进 `docs/BRAIN_HISTORY.md` 或 `.memory`

## 4. 代码策略

- 不制造第二真相
- 不引入胶水层掩盖结构问题
- 优先收敛边界，再扩展功能
