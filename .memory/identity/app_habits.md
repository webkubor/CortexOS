# App 使用习惯

> **作用**: 记录用户在主力设备上的软件使用方式、协作偏好与工作节奏，帮助 Agent 更自然地融入现有工作流。
> **最后更新**: 2026-03-13
> **说明**: 本文件区分“已确认事实”和“从当前配置推断”，避免把推断误写成硬事实。

## 1. 已确认事实

- **知识库核心**: 使用 Obsidian 管理长期记忆库，Vault 位于 `~/Documents/memory`。
- **内容介质**: Markdown 优先。
- **交流语言**: 中文为主，技术术语可英文。
- **代码偏好**: 简洁、可维护、去 AI 味；反对胶水代码、垃圾代码和过度工程化。
- **分享发布标准 (Publishing 2.0)**: 
  - **YAML 配置**: 必须在 Markdown 最顶部包含主题配置（如 `theme: serene-rose`）。
  - **首屏 Hook**: 在 YAML 之后直接挂载公网 URL 版的封面图。
  - **全量图床**: 产出分享稿前，必须自动调用 `r2-uploader` 将本地图片同步至 R2，并替换为 `https://img.webkubor.online/` 链接。
  - **成品逻辑**: 保持“单选标题 + 专业术语对齐 + 扁平化架构拆解”的掘金风格。
- **通知时段**: 10:00 - 20:00。

## 2. 从当前配置推断

- **Obsidian 工作方式**: 偏向文件树、搜索、书签与关系图谱并用，而不是只靠单一入口。
- **知识组织方式**: 倾向把知识、运维、开发规范、发布素材与 secrets 分层管理在同一个 Vault 中。
- **浏览器工作流**: 明显重视 Chrome 会话接管、Chrome DevTools MCP 与浏览器自动化协作。
- **命令行使用**: 当前环境以 zsh 为默认 shell，CLI/Agent 协作占比高。

## 3. 协作建议

- **知识写入**: 可复用经验、复盘、方案与素材优先写入 `~/Documents/memory/knowledge/refined/`。
- **分享写入**: 技术分享内容优先写入 `~/Documents/memory/shares/` (格式: `YYYY-MM-DD-主题.md`)。
- **控制层写入**: 运行日志、任务计划、persona 与状态控制优先写入 `~/Documents/CortexOS/.memory/`。

- **输出风格**: 默认优先高信息密度、少废话、可长期维护的文档与代码。

## 4. 待确认项

- **主力编辑器**: 待确认是 Codex / Cursor / 其他编辑器中的哪一个为第一主力
- **通知方式**: 待确认更偏好系统通知、飞书通知还是对话内提醒
