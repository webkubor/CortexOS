# 🧬 Skill: Project Evolution (项目进化与重塑)

> **“让项目从‘能用’进化到‘专业且高级’。”**

本技能定义了对现有项目进行全量更名、品牌重构、UI 动效注入以及标准化包装的完整流程。

---

## 📋 流程 SOP：五步重塑法

### 第一步：骨髓级路径审计 (Physical Path Audit)
不仅是文件夹更名，必须执行全盘扫描：
- [ ] **全局配置修正**：检查 `~/.gemini/GEMINI.md` 或 `~/.zshrc` 中的物理路径引用（SSOT）。
- [ ] **Git 仓库同步**：使用 `gh repo rename` 修正远程仓库名，同步更新 `git remote set-url`。
- [ ] **环境变量清洗**：批量搜索 `AI_COMMON` 等旧标记，替换为 `CORTEXOS_BRAIN` 等新标记。

### 第二步：认知链条重连 (Context Re-linking)
确保所有 Agent 在启动瞬间即接入新环境：
- [ ] **引导文件更新**：修正 `GEMINI.md` 中的 `Root` 和 `SSOT` 路径。
- [ ] **脚本标记纠偏**：更新 `.zshrc` 里的 Alias 启动脚本和备份文件后缀。

### 第三步：动效与灵魂注入 (Vibe & UI Injection)
拒绝死板的静态界面，让看板“活”起来：
- [ ] **状态感知逻辑**：引入 `Busy` (工作中) 与 `Idle` (挂起) 的二元对立逻辑。
- [ ] **赛博脉动动效**：为活跃节点添加呼吸灯、流光条、以及斜纹流动进度条。
- [ ] **队长核心特效**：为 0 号机节点增加流光边框，强化指挥感。

### 第四步：客观进度背书 (Objective Progress)
杜绝 AI 的进度欺骗，实现基于事实的监控：
- [ ] **TODO.md 强制化**：每个 workspace 根目录必须包含 `TODO.md`。
- [ ] **解析器重构**：更新同步脚本，优先按 `[x]` 勾选比例计算百分比。

### 第五步：主理人级包装 (Professional Packaging)
按照《Vibe 宣言》进行文案深度重塑：
- [ ] **README 门户化**：居中 Logo、Badge 阵列、MCP 工具矩阵表格、快速接入 JSON 配置。
- [ ] **About 哲学化**：不再是简历，而是“外部大脑白皮书”，强调上下文主权与审美信条。

---

## 🛠 触发指令 (Trigger)
当用户提到 **“改名”**、**“包装”**、**“优化 UI”** 或 **“看起来专业点”** 时，Agent 必须立即激活此 SOP。
