# 代码审查输出模板

## 适用范围
用于 AI 或人工对前端改动做 Review，优先发现功能风险、回归风险、类型风险、样式风险和发布风险。

## 输出顺序
固定顺序：
1. 先列问题
2. 再列待确认项
3. 最后才写变更总结

## 标准输出模板
```md
## Findings

1. [严重级别] 文件路径:行号
问题描述。
影响范围。
建议修复方式。

2. [严重级别] 文件路径:行号
问题描述。
影响范围。
建议修复方式。

## Open Questions
- 需要确认的问题 1
- 需要确认的问题 2

## Change Summary
- 本次改动涉及的主要模块
- 已完成的验证
- 仍未覆盖的风险
```

## 严重级别建议
- `High`：会导致功能错误、线上故障、错误金额、错误跳转、严重回归
- `Medium`：边界条件不稳、类型约束缺失、状态处理不完整、兼容性风险
- `Low`：可维护性、注释、命名、风格一致性问题

## 前端审查清单
- **200/500/8 硬阈值检查**：单文件 SFC 是否超过 500 行？Script Setup 是否超过 200 行？State 是否超过 8 个？
- **组件结构检查**：复杂组件是否已执行目录化拆分（`index.vue` + `useLogic.ts` + `styles.scss`）？
- **逻辑抽离检查**：静态配置是否已进入 `constants.ts`？通用逻辑是否已进入 `composables` 或 `utils`？
- 主流程是否可用
- 异常流程是否收口
- 类型边界是否明确
- 是否引入重复请求逻辑
- 是否引入硬编码文案
- 是否破坏移动端布局
- 是否遗漏埋点、国际化、版本记录或回滚信息

## 禁止事项
- 禁止把“个人偏好”当作高优先级问题
- 禁止没有证据就判定有 bug
- 禁止先写总结再写问题
- 禁止漏掉版本记录、注释和发布链路风险
