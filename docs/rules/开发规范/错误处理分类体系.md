# 错误处理分类体系

## 定位
本文档是 [[AI前端开发规范]] 网络层规范的强制补充，定义前端错误的分类、归属层、处理方式和用户提示标准。

AI 在处理任何错误时，必须先判断错误类型，再按对应规则处理，禁止在页面组件中随意 catch 并 console.log。

---

## 错误分类总览

```
前端错误
├── A. 网络层错误（请求未到达业务逻辑）
│   ├── A1. 连接/超时错误
│   └── A2. HTTP 状态码错误（4xx / 5xx）
├── B. 业务层错误（请求成功，业务逻辑拒绝）
│   ├── B1. 通用业务错误（code !== 0/200）
│   ├── B2. 鉴权失效（401 / token 过期）
│   └── B3. 权限拒绝（403 / 无操作权限）
├── C. 数据层错误（响应数据不符合预期）
│   ├── C1. 类型/结构不匹配
│   └── C2. 金额/精度计算异常
└── D. 运行时错误（前端代码执行期间）
    ├── D1. Vue 组件渲染错误
    └── D2. 未处理的 Promise 拒绝
```

---

## A. 网络层错误

**归属层**：`src/utils/request.ts`（请求层统一处理，不得泄漏到业务组件）

### A1. 连接/超时错误

| 错误信号 | 典型场景 |
|---------|---------|
| `TypeError: Failed to fetch` | 无网络、DNS 解析失败 |
| `TimeoutError` / `AbortError` | 请求超时 |
| CORS 错误 | 跨域配置问题（通常仅开发环境） |

**处理规则**：
- 统一在请求层 catch，转换为标准 `AppError` 结构
- 用户提示：使用通用网络错误文案，不暴露技术细节
- 不重试（除非明确配置了幂等接口的重试策略）

```typescript
// 请求层标准处理示例
catch (err) {
  if (err instanceof TypeError || err.name === 'AbortError') {
    throw new AppError('NETWORK_ERROR', '网络连接异常，请检查网络后重试')
  }
}
```

### A2. HTTP 状态码错误

| 状态码 | 含义 | 处理方式 |
|--------|------|---------|
| 400 | 请求参数错误 | 提示"请求参数有误"，记录日志 |
| 401 | 未鉴权 | 见 B2（鉴权失效） |
| 403 | 无权限 | 见 B3（权限拒绝） |
| 404 | 资源不存在 | 提示"资源不存在"，不跳转 |
| 429 | 请求过频 | 提示"操作过于频繁，请稍后重试" |
| 5xx | 服务端异常 | 提示"服务异常，请稍后重试"，上报日志 |

## B. 业务层错误

**归属层**：请求层负责识别和转换，API 层可向上透传，页面组件只展示结果，不直接判断 `code`。

### B1. 通用业务错误

项目中常见的响应结构：
```json
{ "code": 1001, "msg": "订单已关闭", "data": null }
```

**处理规则**：
- 请求层统一解包，当 `code` 不在成功范围内时，抛出 `AppError`
- `AppError` 携带 `code`、`message`、`raw`（原始响应）三个字段
- 页面组件只处理 `AppError.message`，不直接判断 `code` 值

```typescript
// 禁止：页面组件内判断业务 code
const res = await fetchOrder(id)
if (res.code !== 200) { // ❌
  toast(res.msg)
}

// 正确：API 层返回已解包的数据，错误由请求层统一抛出
try {
  const order = await fetchOrder(id)
  // order 已是业务数据，无需判断 code
} catch (err) {
  if (err instanceof AppError) {
    toast(err.message) // ✅
  }
}
```

### B2. 鉴权失效（401 / Token 过期）

**处理规则**（请求层统一处理，严禁分散到页面）：
1. 清除本地 token / session
2. 记录当前路由，用于登录后跳回
3. 跳转登录页，附带 `redirect` 参数
4. 不弹 toast（跳转本身即反馈）

```typescript
// 请求层
if (status === 401) {
  authStore.clearToken()
  router.push({ name: 'Login', query: { redirect: router.currentRoute.value.fullPath } })
  return // 不继续抛错，避免页面重复处理
}
```

### B3. 权限拒绝（403）

**处理规则**：
- 区分"页面级无权限"和"操作级无权限"
- 页面级：由路由守卫拦截，跳转 403 页面
- 操作级：请求层抛出 `AppError('FORBIDDEN', '无操作权限')`，页面展示提示，不跳转

## C. 数据层错误

**归属层**：API 层或组合式函数（composable）内处理，不暴露给模板层。

### C1. 类型/结构不匹配

接口返回的数据结构与 TypeScript 类型定义不符时：

**处理规则**：
- 使用 `zod` 或手写 guard 函数在 API 层校验响应结构
- 校验失败时抛出 `AppError('DATA_SCHEMA_ERROR', '数据格式异常')`
- 同时上报日志（字段名、期望类型、实际值），便于排查接口变更

```typescript
// API 层结构校验示例
import { z } from 'zod'

const OrderSchema = z.object({
  orderId: z.string(),
  amount: z.string(), // 金额以字符串传输，防止精度丢失
  status: z.enum(['PENDING', 'PAID', 'CLOSED']),
})

export async function fetchOrder(id: string) {
  const raw = await request.get(`/order/${id}`)
  const result = OrderSchema.safeParse(raw)
  if (!result.success) {
    reportError('DATA_SCHEMA_ERROR', { id, error: result.error })
    throw new AppError('DATA_SCHEMA_ERROR', '数据格式异常，请联系技术支持')
  }
  return result.data
}
```

### C2. 金额/精度计算异常

**强制规则**（支付场景不可违反）：
- 金额字段必须以**字符串**接收，禁止用 `number` 类型承接金额
- 所有金额运算必须通过 `bignumber.js`，禁止原生 `+` / `-` / `*` / `/`
- 展示前必须做合法性校验：非空、非 NaN、非负数

```typescript
// 禁止
const fee = res.amount * 0.01 // ❌ 浮点精度丢失

// 正确
import BigNumber from 'bignumber.js'
const fee = new BigNumber(res.amount).multipliedBy('0.01').toFixed(2) // ✅
```

## D. 运行时错误

**归属层**：全局错误边界，不由业务组件处理。

### D1. Vue 组件渲染错误

在 `main.ts` 或顶层 App 组件注册全局错误处理：

```typescript
// main.ts
app.config.errorHandler = (err, instance, info) => {
  reportError('VUE_RENDER_ERROR', { err, component: instance?.$options.name, info })
  // 不弹 toast，渲染错误通常意味着页面已损坏，跳转错误页
  router.push('/error')
}
```

### D2. 未处理的 Promise 拒绝

```typescript
// main.ts
window.addEventListener('unhandledrejection', (event) => {
  reportError('UNHANDLED_PROMISE', { reason: event.reason })
  event.preventDefault() // 阻止控制台红色报错（可选）
})
```

## AppError 标准结构

所有错误统一通过 `AppError` 类传递，禁止直接 throw 原生 `Error` 或字符串：

```typescript
// src/utils/error.ts
export type ErrorCode =
  | 'NETWORK_ERROR'
  | 'HTTP_ERROR'
  | 'BIZ_ERROR'
  | 'FORBIDDEN'
  | 'UNAUTHORIZED'
  | 'DATA_SCHEMA_ERROR'
  | 'CALC_ERROR'
  | 'VUE_RENDER_ERROR'
  | 'UNHANDLED_PROMISE'

export class AppError extends Error {
  constructor(
    public code: ErrorCode,
    message: string,
    public raw?: unknown,
  ) {
    super(message)
    this.name = 'AppError'
  }
}
```

## 用户提示规范

| 错误类型 | 提示策略 | 文案原则 |
|---------|---------|---------|
| 网络/超时 | toast，3s 消失 | 说明现象，给出用户可操作的建议 |
| 业务错误 | toast 或内联提示 | 直接用接口返回的 `msg`，AI 不改写 |
| 鉴权失效 | 跳转，不 toast | 无需文案，跳转即反馈 |
| 权限拒绝 | 内联提示 | "暂无此操作权限" |
| 数据异常 | toast | "数据格式异常，请联系技术支持" |
| 金额异常 | 阻断操作，内联提示 | "金额计算异常，请刷新后重试"，禁止继续流程 |
| 渲染错误 | 跳转错误页 | 错误页展示"页面异常，请刷新" |

**禁止**：
- 将技术错误信息（stack trace、接口路径、字段名）直接展示给用户
- 金额相关异常静默处理或只打 log，必须阻断后续操作

## 错误上报规范

```typescript
// src/utils/report.ts
export function reportError(code: ErrorCode, context: Record<string, unknown>) {
  // 接入团队现有监控系统（Sentry / 自研 / 日志平台）
  // 至少记录：errorCode、页面路由、用户标识（脱敏）、时间戳、上下文
  console.error(`[${code}]`, context) // 本地开发降级为 console
}
```

上报必须包含的字段：

| 字段 | 说明 |
|------|------|
| `errorCode` | AppError.code |
| `route` | 当前路由 fullPath |
| `userId` | 脱敏用户标识（如有） |
| `timestamp` | ISO 格式时间 |
| `context` | 错误相关上下文（接口路径、参数、响应摘要） |
