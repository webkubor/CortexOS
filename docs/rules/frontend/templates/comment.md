# 函数头注释模板

## 适用范围
用于导出函数、组合式函数、请求封装、业务处理函数、关键工具函数。

## 最小模板
```ts
/**
 * 用途：一句话说明这个函数解决什么问题。
 * 参数：说明关键参数的业务含义。
 * 返回：说明返回值结构或返回语义。
 * 副作用：说明是否会发请求、写缓存、跳转、埋点、改全局状态。
 * 边界：说明异常场景、空值、默认值、兼容逻辑。
 */
```

## 通用函数模板
```ts
/**
 * 用途：根据订单状态生成前端展示文案。
 * 参数：
 * - status: 后端返回的订单状态码。
 * - locale: 当前语言标识。
 * 返回：前端直接展示的状态文案。
 * 副作用：无。
 * 边界：未知状态码时返回兜底文案，避免页面显示空字符串。
 */
export function getOrderStatusLabel (status: string, locale: string): string {
  // ...
}
```

## 异步请求函数模板
```ts
/**
 * 用途：获取订单详情并统一前端可消费的数据结构。
 * 参数：
 * - orderId: 订单 ID。
 * 返回：标准化后的订单详情对象。
 * 副作用：会触发网络请求；错误时可能抛出异常或进入统一报错流程。
 * 边界：当订单不存在、接口超时或返回结构异常时，按请求层约定处理。
 */
export async function fetchOrderDetail (orderId: string): Promise<OrderDetail> {
  // ...
}
```

## 组合式函数模板
```ts
/**
 * 用途：封装支付提交流程中的加载态、结果态和异常态切换。
 * 参数：
 * - orderId: 当前订单 ID。
 * 返回：支付流程相关的状态、方法和计算属性。
 * 副作用：会发请求、写本地状态，并可能触发页面跳转。
 * 边界：重复提交、接口失败、订单超时等场景必须在注释中提前说明。
 */
export function usePaymentSubmitFlow (orderId: string) {
  // ...
}
```

## 文件头注释模板
```ts
/**
 * 模块职责：统一处理订单请求的协议兼容、错误收口和类型输出。
 * 使用边界：页面和组件只能通过这里暴露的函数拿订单数据，不直接发请求。
 */
```

## 禁止事项
- 禁止写“获取数据”“处理逻辑”这类空话注释。
- 禁止注释和实际行为不一致。
- 禁止只有参数名、没有业务含义说明。
- 禁止高业务密度函数完全无注释。
